# Create Merge Request for E2E Test

Create a GitLab MR for a completed and passing E2E test.

## Usage

```
/qa-create-mr OXDEV-123
```

## Prerequisites

- `memory/tickets/$ARGUMENTS/triage.json` must exist
- `memory/tickets/$ARGUMENTS/test-results.json` must exist with `status: "passed"`
- `memory/tickets/$ARGUMENTS/implementation.md` must exist
- Code must be committed and pushed to a feature branch

## Process

### Step 1: Validate Prerequisites

1. Read `memory/tickets/$ARGUMENTS/test-results.json`
   - If `status` is NOT "passed", STOP: "Tests are not passing. Run `/qa-implement-test $ARGUMENTS` first."
2. Read `memory/tickets/$ARGUMENTS/implementation.md` for branch name and file list
3. Read `memory/tickets/$ARGUMENTS/triage.json` for ticket metadata

### Step 2: Verify Branch State

```bash
cd framework/
git status
git log --oneline -5
```

- Verify all changes are committed
- Verify branch is pushed to remote
- Verify branch name matches: `test/$ARGUMENTS-<slug>`

### Step 3: Create GitLab MR

Use the `glab` CLI to create the merge request:

```bash
glab mr create \
  --source-branch "test/$ARGUMENTS-<slug>" \
  --target-branch "developmentV2" \
  --title "test(<feature_area>): $ARGUMENTS - <summary>" \
  --description "$(cat <<'EOF'
## Summary
- Add E2E test for <feature_area>: <summary>
- Test covers: <target_pages>
- Complexity: <complexity>

## Changes
- `tests/UI/<feature_area>/<testName>/<testName>.test.js` -- new test file
- `actions/<feature_area>.js` -- <new/updated> action functions
- `selectors/<feature_area>.json` -- <new/updated> selectors

## Test Results
- Total: <N> | Passed: <N> | Failed: 0
- Environment: <env>
- Debug cycles: <N>

## Ticket
$ARGUMENTS

---
*Generated by QA E2E Agent*
EOF
)"
```

Capture the MR URL from the output.

### Step 4: Add Jira Comment

Add a comment on the Jira ticket with the MR link:

```
**[QA Agent: pr-agent]** <timestamp>

**Merge Request Created**

| Field      | Value               |
|------------|---------------------|
| MR         | <MR-URL>            |
| Branch     | `test/$ARGUMENTS-<slug>` |
| Target     | `developmentV2`     |
| Tests      | <N> passed          |
```

### Step 5: Write PR Result

Write to `memory/tickets/$ARGUMENTS/pr-result.md`:

```markdown
# PR Result: $ARGUMENTS

- **MR URL**: <MR-URL>
- **Source branch**: test/$ARGUMENTS-<slug>
- **Target branch**: developmentV2
- **Created**: <ISO-8601>
- **Test results**: <N> passed, 0 failed
- **Files changed**: <list>
```

### Step 6: Update Labels

```bash
acli jira workitem edit --key "$ARGUMENTS" --remove-labels "ai-in-progress" --yes
acli jira workitem edit --key "$ARGUMENTS" --labels "ai-done" --yes
```

### Step 7: Update Checkpoint

Update `memory/tickets/$ARGUMENTS/checkpoint.json`:
- Add "pr" to `completed_stages`
- Set `status` to "completed"

### Step 8: Append Audit Log

Append to `memory/tickets/$ARGUMENTS/audit.md`:
```
### [<ISO-8601>] pr-agent
**Action**: Create MR for $ARGUMENTS
**Target**: <MR-URL>
**Result**: MR created successfully
**Details**: Branch test/$ARGUMENTS-<slug> -> developmentV2
```

### Step 9: Report to Dashboard

```bash
./scripts/report-to-dashboard.sh $ARGUMENTS pr
```

### Step 10: Clean Up Session Data

Remove playwright-cli session files (console logs, page snapshots) accumulated during browser exploration:

```bash
rm -rf .playwright-cli/ 2>/dev/null || true
```

## Arguments

- `$ARGUMENTS` -- the Jira ticket key (e.g., OXDEV-123)
