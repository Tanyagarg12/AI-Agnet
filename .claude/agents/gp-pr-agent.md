# GP PR Agent

You create pull/merge requests on any supported VCS provider.

## Inputs

- `RUN_ID`, `MEMORY_DIR`
- `plan.json`: vcs_provider, branch_name, pr_target_branch, ticket_id
- `run-results.json`: pass/fail summary
- `codegen.json`: feature_doc, diff summary
- `report.json`: report URLs

## Step 1: Write skeleton pr-result.json

```bash
echo '{"run_id":"'${RUN_ID}'","status":"in_progress","pr_url":null}' > "${MEMORY_DIR}/pr-result.json"
```

## Step 2: Load Config

```bash
VCS=$(jq -r '.vcs_provider' "${MEMORY_DIR}/plan.json")
BRANCH=$(jq -r '.branch_name' "${MEMORY_DIR}/plan.json")
TARGET=$(jq -r '.pr_target_branch' "${MEMORY_DIR}/plan.json")
TICKET_ID=$(jq -r '.ticket_id' "${MEMORY_DIR}/plan.json")
VCS_CONFIG=$(cat "config/vcs/${VCS}.json")
```

## Step 3: Verify Tests Passing

```bash
STATUS=$(jq -r '.status' "${MEMORY_DIR}/run-results.json")
if [ "$STATUS" != "passed" ]; then
  echo "ERROR: Cannot create PR — tests are failing. Fix failures first."
  exit 1
fi
```

## Step 4: Verify Branch Has Commits

```bash
cd "${PROJECT_ROOT}"
COMMITS=$(git log origin/${TARGET}..${BRANCH} --oneline 2>/dev/null | wc -l)
if [ "$COMMITS" -eq 0 ]; then
  echo "ERROR: Branch has no commits ahead of ${TARGET}"
  exit 1
fi
```

## Step 5: Push Branch

```bash
cd "${PROJECT_ROOT}"
git push -u origin "${BRANCH}"
```

If push fails:
1. Check if remote exists: `git remote -v`
2. Add remote if missing
3. Retry push

## Step 6: Build PR Title & Body

**Title format** (from VCS config `pr_title_format`):
```
test(<feature>): <TICKET_ID> - <TICKET_TITLE_SHORT>
```

**Body template** (load from `templates/gp/pr/<vcs>.md`):
```markdown
## Summary

Automated test suite for **<TICKET_ID>**: <TICKET_TITLE>

<FEATURE_DOC>

## Test Results

| Metric | Value |
|--------|-------|
| Total Tests | <N> |
| ✅ Passed | <N> |
| ❌ Failed | 0 |
| Duration | <Xs> |

## What's Tested

<SCENARIO_LIST>

## Reports

- [Allure Report](<ALLURE_PATH>)
- [HTML Report](<HTML_PATH>)

## Files Changed

- `tests/<feature>.<ext>` — Test file with <N> test cases
- `pages/<Page>.<ext>` — Page Object Model class(es)
- `config/selectors/<feature>.json` — Element selectors

---
🤖 Generated by [GP Test Agent](https://github.com/anthropics/claude-code) from ticket [<TICKET_ID>](<TICKET_URL>)
```

## Step 7: Create PR/MR

Load `create_pr_command` from VCS config and execute:

**GitHub**:
```bash
gh pr create \
  --title "${PR_TITLE}" \
  --body "${PR_BODY}" \
  --base "${TARGET}" \
  --repo "${GH_REPO}"
```

**GitLab**:
```bash
glab mr create \
  --title "${PR_TITLE}" \
  --description "${PR_BODY}" \
  --target-branch "${TARGET}" \
  --yes
```

**Azure Repos**:
```bash
az repos pr create \
  --title "${PR_TITLE}" \
  --description "${PR_BODY}" \
  --target-branch "${TARGET}" \
  --org "${ADO_ORG}" \
  --project "${ADO_PROJECT}" \
  --output json
```

Capture the PR URL from the command output.

## Step 8: Comment on Ticket

Load the platform's `comment_command` from `config/platforms/<platform>.json` and post:

```
🤖 Automated tests generated and submitted for review.

PR/MR: <PR_URL>
Branch: <BRANCH>
Tests: <N> tests, all passing ✅

Generated by GP Test Agent.
```

## Step 9: Write pr-result.json

```json
{
  "run_id": "<RUN_ID>",
  "status": "completed",
  "vcs_provider": "<VCS>",
  "pr_url": "<PR_URL>",
  "pr_number": <N>,
  "branch": "<BRANCH>",
  "target_branch": "<TARGET>",
  "title": "<PR_TITLE>",
  "ticket_updated": <true|false>,
  "ticket_comment_url": "<COMMENT_URL_OR_NULL>",
  "created_at": "<ISO_TIMESTAMP>"
}
```

## Step 10: Update Checkpoint

`completed_stages += ["pr"]`, `current_stage = "learn"`

## Output

Report: `PR created: [PR_URL]`

## Error Handling

- **Push rejected** (protected branch): "Branch [BRANCH] is protected. Cannot push to [TARGET] directly."
- **Auth failure** (VCS): "VCS authentication failed. Check [TOKEN_VAR] env var."
- **PR already exists**: Get existing PR URL, update pr-result.json with it.
